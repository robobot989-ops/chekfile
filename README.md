# DXF Double Line Checker

<img src="https://img.shields.io/badge/Python-3.10%2B-3776AB?logo=python&logoColor=white"> <img src="https://img.shields.io/badge/ezdxf-1.4%2B-important"> <img src="https://img.shields.io/badge/shapely-2.0%2B-important"> <img src="https://img.shields.io/badge/license-MIT-blue">

Сервис для автоматического поиска **двойных линий** в DXF-чертежах. Разработан для производителей вырубных штампов (штанцформ) для контроля качества файлов, отправляемых на лазерную резку фанеры.

## Проблема

В DXF-файлах, подготовленных для лазерной резки, часто встречаются двойные линии — когда одна линия накладывается на другую или находится на расстоянии менее **0.1 мм**. Это приводит к:

- двойному резу лазера
- увеличенному времени обработки
- браку фанеры
- износу оборудования

## Решение

Сервис в реальном времени отслеживает появление новых DXF-файлов в целевой папке (`Z:\LASERTECHNO\<год>\<месяц>\<день>`), автоматически проверяет их и генерирует наглядный HTML-отчёт с подсветкой проблемных мест.

## Возможности

- **Автоматический мониторинг** — watchdog + polling для новых/изменённых файлов
- **Умный анализ** — пространственный R-tree индекс (O(n log n)), кластеризация ошибок
- **Наглядные отчёты** — SVG-рендер чертежа с красной подсветкой проблем
- **Веб-дашборд** — статистика, список проверок, ссылки на отчёты
- **SQLite-память** — не перепроверяет уже проверенные файлы
- **Параллельная проверка** — до 4 файлов одновременно

## Быстрый старт

```bash
pip install -r requirements.txt

# Фоновый мониторинг сегодняшней папки
python main.py

# Однократная проверка
python main.py --once

# Проверить папку за конкретную дату
python main.py --date 2026-06-24

# Проверить произвольную папку
python main.py --path Z:\LASERTECHNO\2026\06\25

# Только веб-дашборд (просмотр результатов)
python main.py --web-only
```

**Веб-дашборд** будет доступен по адресу: [http://localhost:8080](http://localhost:8080)

## Как это работает

1. **Загрузка DXF** — парсинг всех примитивов (LINE, LWPOLYLINE, ARC, CIRCLE, SPLINE, ELLIPSE), аппроксимация кривых отрезками
2. **Пространственный индекс** — построение R-tree (STRtree) для быстрого поиска соседей
3. **Поиск коллизий** — для каждого сегмента поиск соседних в радиусе 0.1 мм, вычисление минимального расстояния
4. **Кластеризация** — группировка близких проблем в одну (connected components, чтобы дуга не порождала десятки ошибок)
5. **Визуализация** — конвертация DXF в SVG, подсветка проблем красным, HTML-отчёт с боковой панелью

## Параметры конфигурации (`config.py`)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `TOLERANCE` | `0.1` мм | Допустимое расстояние между линиями |
| `POLL_INTERVAL` | `30` сек | Период проверки новых файлов |
| `MAX_WORKERS` | `4` | Количество параллельных проверок |
| `WEB_PORT` | `8080` | Порт веб-дашборда |
| `USE_WATCHDOG` | `True` | Включить watchdog реального времени |
