# DXF Double Line Checker

Сервис для автоматического поиска **двойных линий** в DXF-чертежах. Разработан для производителей вырубных штампов (штанцформ) для контроля качества файлов, отправляемых на лазерную резку фанеры.

## Возможности

- Автоматический мониторинг папки (`Z:\LASERTECHNO\<год>\<месяц>\<день>`)
- Поиск двойных линий (расстояние < 0.1 мм) и полных перекрытий
- Пространственный R-tree индекс O(n log n)
- Кластеризация ошибок — дуги не плодят десятки одинаковых проблем
- SVG-визуализация с красной подсветкой проблемных мест
- Веб-дашборд со статистикой и отчётами
- Watchdog реального времени + периодический polling
- Двуязычный интерфейс (RU/EN) — переключение по кнопке в дашборде и в отчёте
- Отправка отчёта на email (SMTP)
- SQLite-память — исключает повторную проверку файлов

## Быстрый старт

```bash
pip install -r requirements.txt

# Фоновый мониторинг сегодняшней папки
python main.py

# Однократная проверка
python main.py --once

# Указать дату или путь
python main.py --date 2026-06-24
python main.py --path Z:\LASERTECHNO\2026\06\25

# Только веб-дашборд
python main.py --web-only

# Указать порт (если 8080 занят)
python main.py --web-port 9090
```

**Веб-дашборд:** [http://localhost:8080](http://localhost:8080) (добавьте `?lang=en` для английского)

## Параметры командной строки

| Аргумент | Описание |
|---|---|
| `--path PATH` | Путь к папке с DXF |
| `--date YYYY-MM-DD` | Дата папки |
| `--once` | Однократная проверка |
| `--web-only` | Только веб-сервер |
| `--web-port PORT` | Порт (по умолч. 8080) |
| `--no-web` | Без веб-сервера |

## Конфигурация (`config.py`)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `TOLERANCE` | `0.1` мм | Допустимое расстояние между линиями |
| `MIN_PROBLEM_DISTANCE` | `0.001` мм | Минимальное расстояние проблемы (фильтр точных наложений) |
| `POLL_INTERVAL` | `30` сек | Период проверки новых файлов |
| `MAX_WORKERS` | `4` | Параллельных проверок |
| `WEB_PORT` | `8080` | Порт веб-дашборда |
| `USE_WATCHDOG` | `True` | Watchdog реального времени |

## Как это работает

1. **Загрузка DXF** — LINE, LWPOLYLINE, ARC, CIRCLE, SPLINE, ELLIPSE → отрезки
2. **Индексация** — R-tree (STRtree) для быстрого поиска соседей
3. **Поиск** — каждый сегмент проверяется с соседями в радиусе 0.1 мм
4. **Кластеризация** — близкие проблемы группируются (connected components)
5. **Отчёт** — SVG с подсветкой, HTML с боковой панелью, кнопками навигации

```
├── monitor/
│   ├── state.py            # SQLite
│   └── watcher.py          # Watchdog + polling
└── web/
    ├── server.py           # HTTP-дашборд + API
    ├── emailer.py          # Отправка email
    └── templates/
        └── report.html     # Шаблон отчёта (RU/EN)
```
