# DXF Double Line Checker

Сервис для автоматического поиска **двойных линий** в DXF-чертежах. Разработан для производителей вырубных штампов (штанцформ) для контроля качества файлов, отправляемых на лазерную резку фанеры.

## Проблема

В DXF-файлах, подготовленных для лазерной резки, часто встречаются двойные линии — когда одна линия накладывается на другую или находится на расстоянии менее **0.1 мм**. Это приводит к двойному резу лазера, увеличению времени обработки, браку фанеры и износу оборудования.

## Возможности

- Автоматический мониторинг папки (`Z:\LASERTECHNO\<год>\<месяц>\<день>`)
- Поиск двойных линий (расстояние < 0.1 мм) и полных перекрытий
- Пространственный R-tree индекс для быстрого анализа (O(n log n))
- Кластеризация ошибок — дуги не порождают десятки одинаковых проблем
- HTML-отчёт с SVG-визуализацией и подсветкой проблемных мест
- Веб-дашборд со статистикой проверок
- Watchdog реального времени + периодический polling
- SQLite-память — исключает повторную проверку одних и тех же файлов

## Быстрый старт

```bash
pip install -r requirements.txt

# Фоновый мониторинг сегодняшней папки
python main.py

# Однократная проверка
python main.py --once

# Указать дату или путь
python main.py --date 2026-06-24
python main.py --path Z:\LASERTECHNO\2026\06\25

# Только веб-дашборд
python main.py --web-only

# Указать порт (если 8080 занят)
python main.py --web-port 9090
```

**Веб-дашборд:** [http://localhost:8080](http://localhost:8080)

## Параметры командной строки

| Аргумент | Описание |
|---|---|
| `--path PATH` | Путь к папке с DXF |
| `--date YYYY-MM-DD` | Дата папки |
| `--once` | Однократная проверка без watchdog |
| `--web-only` | Только веб-сервер |
| `--web-port PORT` | Порт веб-сервера (по умолчанию 8080) |
| `--no-web` | Без веб-сервера |

## Конфигурация (`config.py`)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `TOLERANCE` | `0.1` мм | Допустимое расстояние между линиями |
| `POLL_INTERVAL` | `30` сек | Период проверки новых файлов |
| `MAX_WORKERS` | `4` | Параллельных проверок |
| `WEB_PORT` | `8080` | Порт веб-дашборда |
| `USE_WATCHDOG` | `True` | Watchdog реального времени |

## Как это работает

1. **Загрузка DXF** — парсятся LINE, LWPOLYLINE, ARC, CIRCLE, SPLINE, ELLIPSE; кривые аппроксимируются отрезками
2. **Индексация** — строится R-tree (STRtree) для пространственного поиска
3. **Поиск коллизий** — каждый сегмент проверяется с соседями в радиусе 0.1 мм
4. **Кластеризация** — близкие проблемы группируются через связные компоненты
5. **Отчёт** — DXF конвертируется в SVG, проблемы подсвечиваются красным, генерируется HTML

```
│   └── report.py           # HTML-отчёты

│   ├── state.py            # SQLite

    └── templates/
        └── report.html     # Шаблон отчёта
```
